Sous le capot

Construit avec l'aide de l'IA

Pas de Unity, pas de Godot. ZOMI est écrit en C avec SDL2 et FFmpeg, développé avec l'assistance d'outils d'IA, compilé sur MSYS2 et Linux. La manette ESP32 est un projet à part — du firmware C jusqu'au protocole UDP custom.

Architecture en couches

Une vue rapide de l'organisation interne, du polling de l'input jusqu'au rendu des cinématiques.

1

Entrée

  • ›Polling SDL_Event 60 Hz
  • ›ESP32 — listener UDP dédié sur thread séparé
  • ›PS4 — SDL_GameController + remap automatique
  • ›Clavier — table de scancodes AZERTY native
2

État de jeu

  • ›Machine à états globale (menu / acte / cinématique / pause / mort)
  • ›Inventaire trois slots, vies, score, sauvegarde
  • ›Multi : deux instances de joueur indépendantes
3

Rendu

  • ›Render target par viewport (split-screen)
  • ›Sprite-strips animés (36 colonnes × N lignes)
  • ›Parallax 3 couches (ciel / bâtiments / sol)
  • ›HUD adaptatif selon l'input détecté
4

Audio

  • ›Mixer SDL2_mixer (musique loop + FX one-shot)
  • ›FFmpeg audio extrait des MP4, PTS aligné sur la vidéo
  • ›Boost soft-knee 1.8× pour les dialogues
5

Cinématiques

  • ›Décodeur libavcodec custom (frame YUV → SDL_Texture)
  • ›Lecture en temps réel avec saut d'images si retard
  • ›11 cinématiques entre les actes

Focus

Manette ESP32 — protocole UDP custom

Une carte ESP32, un joystick analogique, quatre boutons d'action, deux gâchettes L1/R1, un bouton de pause. Le firmware envoie en UDP des paquets binaires de 8 octets toutes les 16 ms. Côté jeu, un thread C dédié écoute le port, désérialise et met à jour l'état d'input partagé. Aucun framework — juste socket(), recvfrom(), et la conviction que ça allait fonctionner.
